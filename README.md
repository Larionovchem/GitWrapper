# GitHub Wrapped — План разработки

> Проект на 2-3 дня для портфолио. Цель — показать, что ты умеешь современный продуктовый фронтенд, а не «ещё одну тудушку».

---

## 📋 Обзор

**Что строим:** одностраничник, который берёт ник на GitHub и показывает «Wrapped»-историю по слайдам — топ языки, активность по часам/дням, лучший месяц, общая статистика.

**Финальный стек:**

- Vite + React 18 + TypeScript (`strict: true`)
- Tailwind CSS
- TanStack Query (server state)
- Zod (валидация API)
- React Router
- Framer Motion (анимации)
- Recharts (график активности)
- FSD (архитектура папок)
- Vercel (деплой)

**Опционально (если останется время):**

- html-to-image (скачать слайд как PNG)
- Vercel Serverless Function (прокси к GitHub API с PAT)

**Общее время:** ~22 часа активной работы (3 дня × ~7.5ч)

### Задача 1.2: Изучение GitHub API + Zod-схемы · ⏱ 2ч

**Что делаем:**

- Дёргаем нужные endpoint'ы руками через `curl` или Insomnia/Postman, смотрим формат ответов
- Описываем Zod-схемы под `User`, `Repo`, `Event`

- Сохраняем 2-3 примера ответов в `shared/api/__mocks__/` для разработки без сети

**Endpoint'ы, которые нужны:**

- `GET /users/{username}` — профиль
- `GET /users/{username}/repos?per_page=100&sort=updated` — репозитории

**Технологии:** Zod, GitHub REST API

**Где почитать:**

**Совет:** Не описывай **все** поля ответа — только те, что нужны. Если поля нет в схеме, оно отбросится. Это и есть смысл валидации.

**Definition of done:**
